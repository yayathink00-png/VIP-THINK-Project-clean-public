# VIP THINK Project Clean Public Repository

This repository is a clean public-release workspace prepared from the governance review of `yayathink00-png/VIP-THINK-Project`.

## Public Scope

This clean repository currently includes only:
- public-release governance documentation
- publishing and security checklists
- asset governance rules
- contribution boundary notes
- maintainer confirmation templates

It intentionally does not include source modules, generated assets, raw captures, exports, archives, checkpoint handoffs, private prompts, platform resource identifiers, cookies, session workflows, or unreviewed media.

## Responsibility Boundary

This repository is prepared to reduce future public-release risk. It does not certify historical commits, historical public exposure, forks, clones, screenshots, caches, or third-party redistribution from the original repository.

## Excluded By Default

The following remain excluded unless a maintainer explicitly approves them in a future pull request:
- raw data and raw source material
- generated images, videos, audio, and AI intermediate outputs
- ZIP, PDF, HTML, CSV, spreadsheet, slide, and document exports
- dated checkpoint or handoff folders
- private prompt packs and source-media breakdowns
- cookies, session files, tokens, API keys, webhooks, app tokens, table IDs, spreadsheet tokens, and private URLs
- user, customer, student, parent, supplier, order, contract, pricing, revenue, cost, account, or internal strategy data

## Release Process

Before making this repository public or adding project files:
1. Confirm the exact public scope with the maintainer.
2. Copy only approved whitelist files.
3. Run the security and file-type scans in `docs/governance/SECURITY_CHECKLIST.md`.
4. Complete `docs/governance/MAINTAINER_CONFIRMATION_TEMPLATE.md`.
5. Use pull requests for all additions.

No force push, visibility change, remote deletion, or history rewrite should be performed without explicit written authorization.
