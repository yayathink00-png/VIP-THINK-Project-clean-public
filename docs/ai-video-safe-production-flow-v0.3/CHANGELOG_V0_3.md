# AI 视频安全生产流 V0.3｜变更记录

版本日期：2026-05-22

## V0.3 的核心变化

| 模块 | V0.2 问题 | V0.3 固化 |
|---|---|---|
| Dreamina 对话 | 把“对话名”当成启动输入，但新对话一开始没有名称 | 新增 Dreamina Dialogue Gate：先建对话，放入内容，再重命名，再绑定 |
| 需求确认 | 生成脚本前容易漏问硬要求 | Requirement Gate 明确确认真人、台湾腔、繁体、竖版、生成方式、源视频用途 |
| 提示词 | 长 prompt 容易 pre-TNS 或生成发散 | 默认短提示词，单段只保留核心画面、动作、台词和参考锚点 |
| 口音 | “台湾腔国语画外音”等写法不准确 | 固定按用户原话写“台湾腔” |
| 人物稳定 | 纯文生视频容易换人 | 固化 Continuity Gate：通过段提取人物参考图，后续优先 image2video |
| 路径 | 相对路径可能导致参考素材失败 | 提交前统一解析为绝对路径 |
| 审核 | 自动复核 blocking 容易被误解为最终失败 | 复核分层，自动结果只做风险提示，最终以人工审片为准 |
| 积分 | 采用和试错混在一起 | 新增 Cost Gate，分开记录最终采用积分和试错积分 |
| 最后一段 | 最后一段通过后仍可能准备下一段 | 最后一段人工通过后直接进入 Final Gate |
| 交付 | 复盘文档不够适合外部移交 | 增加可读复盘模板、故障手册和整包交付结构 |

## 安全版说明

安全版不包含内部脚本快照、真实项目 submit_id、本地文件路径和完整案例明细。以下经验已抽象成通用规则：

- 复杂长 prompt 会增加失败概率。
- 人物变形时，不要继续纯文生视频硬试；应使用上一段已通过人物做参考图。
- “台湾腔”比扩写成复杂音频描述更稳定。
- 每次重提前必须重新展示 prompt 和积分，不可默认继续扣点。
