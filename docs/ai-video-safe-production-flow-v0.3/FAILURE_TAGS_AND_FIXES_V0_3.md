# AI 视频生产流 V0.3｜失败标签与修复手册

失败要先分类，再修 prompt 或流程。不要看到失败就直接重提同一版。

| 标签 | 典型表现 | 优先修复 | 重提前检查 |
|---|---|---|---|
| `PRE_TNS_FAIL` | 平台生成前拦截，未产出视频 | 精简提示词，减少促销词、否定词、过度承诺 | Prompt 不超过必要信息；CTA 只保留台词必要表达 |
| `CHARACTER_DRIFT` | 老师/主角换脸、换年龄、换衣服明显 | 用已通过片段的人物清晰截图，改 image2video | 参考图是绝对路径；人物脸部清晰；裁掉干扰 |
| `VO_ERROR` | 旁白漏词、错词、口音不对、多人混说 | 缩短台词，写唯一旁白，口音按用户原话写 | 不写“台湾腔国语画外音”等混合表述 |
| `TEXT_ARTIFACT` | 屏幕文字、字幕、白板内容乱码或抢戏 | 减少画面文字描述，字幕交给后期 | Prompt 不要求模型生成复杂文字 |
| `LOGO_RISK` | 出现平台 Logo、水印、疑似品牌标 | 裁切参考图，避免边角标识，后期检查 | 参考图边角干净；Stable File Gate 抽帧检查 |
| `PATH_ERROR` | 参考素材上传失败或找不到文件 | 把相对路径转绝对路径 | 本地脚本能解析路径；文件存在 |
| `REFERENCE_OVERLOAD` | 参考太多导致画面混乱或卡失败 | 只保留最关键 1-2 个参考 | 明确哪个参考管人物，哪个参考管构图 |
| `DURATION_WASTE` | 台词太少但生成过长，或动作拖沓 | 缩短时长或增加明确动作节奏 | 本段目标与时长匹配 |
| `PROCESS_BREAK` | 已通过段落被误标阻断，或最后一段还准备下一段 | 修正 gate 状态，不重提视频 | 检查 `gate_status.json` 的 next_action |
| `COST_OVERSPEND` | 试错积分快速上升 | 暂停重提，先人工定位失败标签 | Cost Gate 重新确认预算 |
| `DIALOGUE_NAME_MISSING` | 新对话无法在历史里稳定识别 | 先放安全占位内容，再重命名 | 当前对话名已显示且可选中 |
| `WRONG_DIALOGUE_SELECTED` | 提交到了旧项目对话 | 重新选择本项目对话并绑定 Name Gate | 对话名、session id/url 与本地一致 |

## 修复顺序

1. 先判断是流程问题还是生成质量问题。
2. 流程问题优先修流程，通常不计有效试错。
3. 生成质量问题才改 prompt 并重提。
4. 每次重提前都要重新过 Segment Prompt Gate。
5. 每次重提都要重新等用户回复确认语。

## 常用修复句式

人物连续：

```text
参考图中的同一位女老师继续出现，保持相同年龄、发型、服装气质和亲切表情。
```

台湾腔：

```text
旁白用台湾腔，自然亲切。
```

短 CTA：

```text
旁白：直接点击影片下方链接，为你的孩子报名吧。
```

画面文字降风险：

```text
画面重点是老师和孩子互动，不强调屏幕上的可读文字。
```
