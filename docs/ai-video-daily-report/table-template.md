# AI 视频日报台账填写模板

表格一行对应一条项目或视频的日报数据。只记录有问题、有返工、有人工判断价值的内容；没有问题的归因项不要硬写。

## 必填字段

| 字段 | 填写说明 |
| --- | --- |
| 日期 | 生产日期，格式建议 `YYYY-MM-DD`。 |
| 项目/视频 | 项目名或视频名。 |
| 完成视频数 | 当天完成的整条视频数量。 |
| 处理Segment数 | 当天处理的 Segment 总数。 |
| 处理Segment明细 | Segment 编号，例如 `1、2、3、4`。 |
| 一次通过数 | 首次有效生成后直接采用的 Segment 数。 |
| 一次通过明细 | 对应 Segment 编号。上传失败但未扣费、重试后同策略成功的，不算返工。 |
| 返工数 | 需要重新生成或明显调整策略的 Segment 数。 |
| 返工明细 | 对应 Segment 编号。 |
| 废弃结果数 | 已生成但未采用的结果数。 |
| 废弃结果说明 | 简述废弃原因。 |
| 采用结果数 | 最终采用的生成结果数。 |
| 有效生成结果数 | 实际生成出结果的次数，不含未扣费的上传失败。 |
| 即梦链路问题次数 | 上传、提交、服务异常等链路问题次数。 |
| 即梦链路说明 | 只写真实发生的问题，例如上传 bad gateway。 |
| 生成效果问题数 | 画面、人物、动作、连续性、质感等生成结果不达标次数。 |
| 生成效果说明 | 只写需要复盘的效果问题。 |
| 参考图策略问题数 | 尾帧、首帧、三视图、参考素材策略导致需要调整的次数。 |
| 参考图策略说明 | 说明调整前后策略，例如去掉上一段尾帧后采用。 |
| 播报确认 | 人工确认后填 `是`。 |
| 是否已播报 | 机器人正式发群后填 `是`。 |
| 播报时间 | 正式播报时间。 |
| 备注 | 补充说明。 |

## 可选字段

| 字段 | 填写说明 |
| --- | --- |
| AI执行内容 | AI 实际承担的步骤，例如拆分 Segment、生成提示词、提交即梦、整理日报。 |
| AI实现质量 | 对 AI 输出质量的判断，例如可直接采用、需人工修正、需重跑。 |
| 人工Gate节点 | 人工把关发生在哪些节点，例如提示词确认、参考图确认、积分确认、最终采用确认、播报确认。 |
| 人工判断结果 | 人工最后怎么判断，例如采用、重跑、改参考图、暂停播报。 |
| 人工调整动作 | 人工做了什么调整，例如删除尾帧、换参考图、改提示词、要求重剪。 |

## 归因口径

- 上传资源阶段 bad gateway 归为即梦链路问题。
- 使用上一段尾帧后效果不理想，优先归为生成效果问题；如果根因是参考图策略选择错误，再同步计入参考图策略问题。
- 只播报问题项；素材问题、人工操作问题等为 0 时不要写进日报正文。
- 上传失败但未扣费、未产生有效结果时，不计入有效生成结果数，也不计入返工。
