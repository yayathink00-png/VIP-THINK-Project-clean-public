# 给同事 Codex 的台账自动填写提示词

把下面这段发给同事的 Codex，让它按实际制作记录填写钉钉表格。

```text
你要帮我填写《AI 视频日报台账》的一行数据。请先根据当天制作日志、即梦提交记录、采用/废弃结果、人工确认记录，提取真实数据；不确定的字段留空或标注“待确认”，不要编造。

填写口径：
1. 一行对应一个项目/视频的一天生产记录。
2. 日期使用生产日期，格式 YYYY-MM-DD。
3. 周一播报上周五的数据；其他工作日播报前一天的数据。
4. 一次通过：首次有效生成后直接被采用。上传失败但未扣费、重试后同策略成功，不算返工。
5. 返工：已生成结果未采用，或需要明显调整提示词、参考图、尾帧/首帧策略后重跑。
6. 废弃结果：已生成但最终未采用的结果。
7. 有效生成结果数：实际生成出结果的次数，不含未扣费的上传/提交失败。
8. 即梦链路问题：上传、提交、服务 bad gateway、接口异常等链路故障。
9. 生成效果问题：画面、人物、动作、连续性、质感、口型等生成结果不达标。
10. 参考图策略问题：参考图、首帧、尾帧、三视图、上一段尾帧引用等策略导致效果问题。
11. 只记录有问题、有返工、有人工判断价值的内容；没有问题的项不要硬写说明。
12. AI执行内容要写 AI 做了什么，不写泛泛的“AI参与”。
13. 人工Gate节点要写人工在哪些环节把关，例如提示词确认、参考图确认、积分确认、生成结果采用确认、最终播报确认。
14. 人工判断结果要写人工最后做出的决定，例如采用、重跑、去掉尾帧、换参考图、不播报。
15. 人工调整动作要写人工实际调整了什么，例如改提示词、删除尾帧、替换参考图、要求重新生成。

请输出一行可直接填入表格的数据，字段顺序如下：
日期｜项目/视频｜完成视频数｜处理Segment数｜处理Segment明细｜一次通过数｜一次通过明细｜返工数｜返工明细｜废弃结果数｜废弃结果说明｜采用结果数｜有效生成结果数｜即梦链路问题次数｜即梦链路说明｜生成效果问题数｜生成效果说明｜参考图策略问题数｜参考图策略说明｜播报确认｜是否已播报｜播报时间｜备注｜AI执行内容｜AI实现质量｜人工Gate节点｜人工判断结果｜人工调整动作
```
